Basic CSS rule

  • A basic CSS rule consists of a selector, a property and a value: 
    selector {
  property: value;
}
  • Example: 
    p {
  color: black;
}
  • → this selects all paragraphs, and sets the text color to black

Organise your CSS (4)

  • The browser can only download one resource at a time over a single HTTP connection
  • The number of parallel HTTP connections is limited by the OS
  • In short: the more resources (CSS files, images, scripts...) your page contains, the slower it loads
  • limit your number of CSS files
  • or, concatenate and compress them before publishing your site
  • or, use a HTTP/2 server

CSS

CSS1 selectors

Tag selector

  • A tag selector selects elements by tagname: 
    p {
  /* styles for all paragraphs */
}

Class selector

  • A class selector selects elements by class attribute: 
    .message {
  /* styles for <... class="message">...</...> */
}
  • A multiple class selector selects elements by multiple class attributes: 
    .message.error {
  /* styles for <... class="message error">...</...> */
}

Id selector

  • An id selector select a single element by its id attribute: 
    #active {
  /* styles for <... id="active">...</...> */
}

Pseudo class selector

  • A pseudo class selector selects elements in a certain state or relation: 
    a:link { /* ... */ }
a:visited { /* ... */ }
a:hover { /* ... */ }
a:active { /* ... */ }
  • → more pseudo class selectors defined in CSS2 (see further)

Pseudo element selector

  • A pseudo element selector selects a part of an element: 
    em::first-letter { /* ... */ }
p::first-line { /* ... */ }
  • → more pseudo element selectors defined in CSS2 (see further)
Actually, the double colon notation was introduced in CSS3 to make a differentiation between pseudo class and pseudo element selectors. For the sake of IE8 and older who does not support this, you can still use the old single colon notation: 
em:first-letter { /* ... */ }
p:first-line { /* ... */ }
For CSS3 and higher pseudo element selectors (which are unsupported by IE8- in any notation) however you should always use the double colon notation.

Combined selectors

  • Different selectors can be combined
  • Tag/class selector example: 
    span.error {
  /* styles for <span class="error">...</span> */
}
  • Tag/class/pseudo selector example: 
    a.topmenu:link { /* ... */ }
a.topmenu:hover { /* ... */ }

a.leftmenu:link { /* ... */ }
a.leftmenu:hover { /* ... */ }

Grouped selectors

  • Different selectors may be grouped in a comma-separated list: 
    input, select, a:hover, p.formpar {
  color: blue;
}
  • This is equal to: 
    input {
  color: blue;
}
select{
  color: blue;
}
a:hover {
  color: blue;
}
p.formpar {
  color: blue;
}

Contextual selector

  • A contextual selector selects elements in a certain context
  • Element nested inside another element: 
    #leftcol p {
  /* styles for p nested in div with id 'leftcol' */
}
  • → many more contextual selectors defined in CSS2 (see further)

CSS

CSS2.1 selectors

Universal selector

  • The universal selector selects all elements: 
    * {
  /* ... */
}
  • → the universal selector is slow, but you won't notice unless you have tons of CSS
...just don't add CPU intensive styles like shadows and animations to every single element in your page, but this is common sense...

More contextual selectors

  • Element directly nested inside another element: 
    header > p {
  /* styles for <p>'s where <header> is direct parent */
}
  • Element directly following another element: 
    h2 + p {
  /* styles for <p>'s directly following a <h2> */
}

More pseudo class selectors (1)

  • CSS2 defines :focus and :first-child: 
    input:focus {
  /* selects every input receiving focus */
}

tr:first-child {
  /* selects the first child of every <tr> */
}

More pseudo class selectors (2)

    → many more pseudo element selectors are defined in CSS3 (see this presentation)

Attribute selectors

  • An attribute selector selects elements by attribute: 
    select[multiple] {
  /* <select> contains multiple attribute */
}
input[type=button] {
  /* <input> type attribute is exactly 'button' */
}
a[title~=flower] {
  /* <a> title attribute contains 'flower' */
}
img[src|=http://www.ikdoeict.be] {
  /* <img> src attribute starts with 'http://www.ikdoeict.be' */
}
  • → more attribute selectors are defined in CSS3 (see this presentation)

CSS

CSS properties

Property list

  • You may group properties with the same selector in a list: 
    p {
  font-size: 10px;
  color: red;
}

Grouped properties

  • Some properties may be grouped in a single values list: 
    body {
  background-image: url(../img/bg.jpg);
  background-repeat: repeat-y;
  background-position: right;
}
  • this is the same as: 
    body {
  background: url(../img/bg.jpg) repeat-y right;
}
  • → the order of property values is important!

Quotes around properties

  • Put quotes around values with spaces and ::before or ::after content: 
    p {
  font-family: Arial, Verdana, "sans serif";
}
p:before {
  content: "→";
}
  • Don't put quotes around other values: 
    body {
  background-image: url(img/bg.jpg);
}

Colors (1)

  • Hexadecimal: 
    p {
  color: #cc0099;
  background-color: #ffffff;
}
  • Hexadecimal shorthand: 
    p {
  color: #c09;
  background-color: #fff;
}

Colors (2)

  • Decimal or percentage notation: 
    p {
  color: rgb(204,0,153);
  background-color: rgb(100%,100%,100%);
}
  • Mnemonic notation: 
    p {
  color: VioletRed;
  background-color: white;
}
  • → mnemonic notation is for wussies
    → more notations exist: hsl, rgba (CSS3)...

Sizes (1)

  • The most used units are px (pixel) and em (relative to font size); we'll elaborate on this in the next presentation
  • Shorthand notation for margins, borders...: 
    img {
  margin-top: 0.5em;
  margin-right: 1em;
  margin-bottom: 0.5em;
  margin-left: 1em;
}
  • is the same as: 
    img {
  margin: 0.5em 1em 0.5em 1em; /* top-right-bottom-left */
}
  • is the same as: 
    img {
  margin: 0.5 1em 0.5em; /* top-horizontal-bottom */
}
  • is the same as: 
    img {
  margin: 0.5 1em; /* vertical-horizontal */
}

Sizes (2)

  • For reasons explained later, it is a good idea to start your CSS with: 
    /* default font size is 16px; set html font size to 10px */
html {
  font-size: 62.5%;
}

/* rest of your CSS; 1rem is now 10px */
body {
  font-size: 1.5rem; /* 15px  */
  padding: 1rem; /* 10px  */
}
h1 {
  font-size: 2.2rem; /* 22px  */
}
...

CSS

Priority and inheritance

Priority

  • Final style is a combination of
    • non-conflicting styles: combination
    • conflicting styles: priority rules
    • undefined styles: browser defaults
  • In case of conflict, priority is determined by
    • selector weight
    • in case of same weight: last one wins
  • Inline always has priority over external and internal styles

Priority by order (1)

  • Example: 
    p {
  font: bold smaller Arial;
  font-family: Courier, Verdana;
}
  • → conflict between Arial and Courier
  • → selector is the same for both properties
  • → last one wins, which is Courier

Priority by order (2)

  • What is wrong with this fragment? 
    a:active {
  color: white;
}
a:hover {
  color: lightblue;
}
  • → active is also hover, so link will be lightblue when active
  • → solution: switch both fragments
  • What is the border color in this fragment? 
    p {
  border-color: red;
  border: 1px solid;
}
  • → black, the default value for border

Priority by weight (1)

  • Example: 
    div#idtest {
  font-size: 12px;
}
div.ctest {
  font-size: 14px;
}
  • This element is selected by both selectors: 
    <div id="idtest" class="ctest">...</div>
  • → the first selector weighs more, so 12px

CSS

Practical notes

Browser support

  • CSS1 (1996):
    • fonts; colors, text, alignment, border, paddings...
    • full support by all modern browsers
  • CSS2 (1998) revised in CSS2.1 (2011)
    • positioning, media types, new font properties
    • error fixing, poorly supported features removed in CSS2.1
    • full support by all modern browsers
  • CSS3 (in development):
    • devided in >50 modules
    • animations, drop shadows, gradients, new selectors...
    • some parts supported by some browsers
      (see caniuse.com or quirksmode.org)

CSS reset

  • Use some kind of CSS reset to wipe out browser default differences. The most basic reset: 
  • * {
  margin: 0;
  padding: 0;
}
  • → or pick your favorite reset at http://www.cssreset.com/

Best practices (1)

  • Page content often comes from different sources (libraries, ads, templates...), so use class selectors rather than id selectors to prevent conflicts: 
  • <div id="wrapper">
  <nav id="top">
    <ul>
      <li id="first">...</li>
      <li>...</li>
      <li>...</li>
    </ul>
  </nav>
  <section id="main">...</section>
</div>

  • <div class="wrapper">
  <nav class="top">
    <ul>
      <li class="first">...</li>
      <li>...</li>
      <li>...</li>
    </ul>
  </nav>
  <section class="main">...</section>
</div>

  • You can cut down on id and class attributes by using contextual selectors:

  ...

  • <div class="wrapper">
