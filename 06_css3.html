Part 3: CSS

Core

Tag & attribute selectors

  • New tag selector: 
    p ~ ul {
  /* ul preceded by a paragraph in the same parent */
}
  • New attribute selectors: 
    input[type="text"] {
   /* attribute 'type' has value 'text' */
}
a[title^="The"] {
   /* attribute 'title' starts with 'The' */
}
a[title*=" is the "] {
   /* attribute 'title' contains ' is the ' */
}
a[title$="end."] {
   /* attribute 'title' ends with 'end.' */
}

New pseudo selectors (1)

  • New pseudo element selector: 
  • p::selection {
  /* portion selected by the user */
}
  • Pseudo class selector: 
  • p:first-of-type {
   /* select the first paragraph of each parent */
}
p:last-of-type {
   /* select the last paragraph of each parent */
}
p:only-of-type {
   /* select paragraph if unique in its parent */
}
:only-child {
   /* select if only child */
}
:only-of-type {
   /* select if only child of its type */
}
:last-child {
   /* select the last child */
}
:empty {
   /* select if empty */
}
:enabled {
   /* select if enabled */
}
:disabled {
   /* select if disabled */
}
:checked {
   /* select if checked */
}
:not(p) {
   /* select element if not a paragraph */
}
Note the double colon in ::selection. This was introduced in CSS3 to differentiate between pseudo-elements(::), which select a part of an element, and pseudo-classes(:), which select elements as a whole. For pre-CSS3 pseudo-elements both notation will be allowed, but the new notation is preferred (::first-letter, ::before etc...). Internet Explorer < 9 is a dealbreaker though, as usual..

New pseudo selectors (2)

  • New nth-child pseudo class selector: 
  • /* alternates every 3 rows */
p:nth-child(3n+1) {
   background-color: #aaa;
}
p:nth-child(3n+2) {
   background: #ccc;
}
p:nth-child(3n) {
   background: #eee;
}

New pseudo selectors (3)

  • Form specific pseudo class selector: 
  • :checked {
   /* select checked inputs */
}
:enabled {
   /* select enabled inputs */
}
:disabled {
   /* select disabled inputs */
}
:focus {
   /* input has focus */
}
:required {
   /* input is required */
}
:optional {
   /* input is optional */
}
:in-range {
   /* input value is in its defined range */
}
:out-of-range {
   /* input value is out of its defined range */

:read-only {
   /* read only inputs */
}
:read-write {
   /* not read only inputs */

   /* valid inputs */
}
:invalid {
   /* invalid inputs */
}

Box model — sizing

  • Use a different model for width and height: 
  • .box {
  border: 10px solid #999;
  padding: 20px;
  box-sizing: content-box; /* change to border-box */
  width: 50%;
}

Box model — Flexbox (1)

  • Flexbox is a new set of styles to align elements within a box:
    • display (set to box to induce the Flexbox)
    • box-orient (horizontal or vertical alignment line for children)
    • box-pack (how to spread the children: start, end, center, justify)
    • box-align (alignment along the line: start, end, center, baseline, stretch)
    • box-flex (relative size of a child)
  • It is still too early to use flexbox, but support is growing
  • see http://www.html5rocks...quick/ for more information the specs have changed, check this article instead

Box model — Flexbox (2)

  • Orientation and relative child sizes (Chrome & Safari): 
  • /* for other browsers prefix with -moz-, -o- or -ie- */
#boxcontainer {
  display: -webkit-box;
  -webkit-box-orient: horizontal; /* also try vertical */
}
#boxcontainer .one, #boxcontainer .three {
  -webkit-box-flex: 1;
}
#boxcontainer .two {
  -webkit-box-flex: 2;
}

Box model — Flexbox (3)

  • display: -webkit-box;
